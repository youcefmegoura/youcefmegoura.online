Task: Conduct a Technical Review

Your task is to perform a complete technical review of both websites.

1. User Experience (UX/UI)

Analyze the following aspects:

Evaluate the ease of use of the website.

Assess the navigation between pages.

Analyze the clarity of the interface and design.

Check whether the website is responsive (works well on mobile, tablet, and desktop screens).

Suggest practical improvements and justify them with clear reasons.

2. Performance

Evaluate the performance of the websites:

Measure page loading speed.

Analyze the size of resources (JavaScript, CSS, images).

Identify possible performance optimizations such as:

Lazy Loading

Caching

File compression

Mention the tools used for measurement, such as:

Lighthouse

PageSpeed Insights

Browser DevTools

3. Technical Aspects (Frontend)

Analyze the visible part of the website:

Study the frontend structure.

Evaluate the quality of the code (if visible).

Check technical SEO practices.

Evaluate Accessibility, such as:

alt attributes for images

keyboard navigation

color contrast

4. Technical Aspects (Backend) – if possible

Using the Network tab in browser developer tools:

Observe the behavior of APIs.

Analyze the structure of requests and responses.

Check basic security aspects such as:

HTTP headers

authentication mechanisms (auth)

5. Security

Perform a basic security check:

Verify the following:

The website uses HTTPS.

Forms are properly protected.

No sensitive information is exposed in the frontend.

Provide recommendations for improving security.

6. Technical SEO

Analyze the website’s SEO structure:

Check the structure of headings (H1, H2, H3…).

Verify the presence of meta tags.

Analyze how site speed affects SEO.

Check whether the site is easily indexable by search engines.

Report Submission Format

The report must include:

A short introduction

Observations organized by category

Identified issues

Suggested improvements

An overall platform rating (out of 10)
